title: Deprecation Policy
description: Versioning, compatibility, and sunset rules
---

AssetArcade versions the API by path prefix, such as `/v1/` and `/v2/`.

## Current status

`/v1/` is active and has no planned sunset date.

## Compatibility rules

Non-breaking changes may be added to a live version. Examples include:

- new optional response fields
- new endpoints
- new optional query parameters

Breaking changes require a new major version.

## What counts as a breaking change

Examples include:

- removing or renaming a response field
- changing the type of an existing field
- removing an endpoint
- changing a path or parameter name
- making an optional field required

## Deprecation process

When a new major version is introduced:

1. the new version launches alongside the existing one
2. the older version receives at least 6 months of deprecation time
3. deprecated endpoints are signaled with `Deprecation` and `Sunset` headers
4. deprecated endpoints are marked in the OpenAPI spec

## Versioning rule

AssetArcade versions the API as a whole. Individual endpoints are not versioned separately.
