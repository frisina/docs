## Why AssetArcade exists

AI-assisted game development needs machine-readable assets, not just storefront copy. AssetArcade exposes manifests, context, integration guidance, branches, versions, and publisher data in a form that developers, tools, and AI agents can act on directly.

## Who it's for

- AI coding agents that need structured asset discovery and implementation context
- tool builders integrating marketplace discovery into assistants, IDEs, and pipelines
- developers and publishers who want assets to be understandable inside AI-assisted workflows

## Base path

All endpoints are served under:

`/api/v1`

## Current surface

The current API includes:

- agent search
- asset detail
- manifest, context, and integration endpoints
- capabilities taxonomy
- branch listing and version history
- entitlement checks
- publisher profiles

## How to navigate

Start with [Quickstart](/quickstart) if you want the shortest path from search to implementation. Use the generated Reference section when you need the full request and response surface by tag.

## Versioning

AssetArcade currently serves `/v1/`.

Breaking changes are introduced through a new major version rather than changing the live version in place. See the [Deprecation Policy](/deprecation-policy) for the compatibility rules.
