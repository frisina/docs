AssetArcade helps publishers make game assets easier to discover, evaluate, and integrate through structured metadata, compatibility information, and implementation guidance. The API is designed for developers, tools, and AI-assisted workflows that need more than storefront copy to understand how an asset fits into a project.

## Why AssetArcade exists

Modern development workflows increasingly rely on structured context, not just screenshots and marketing descriptions. AssetArcade exposes manifests, integration guidance, branches, versions, and publisher data in a format that helps assets move more cleanly from discovery to implementation.

## Who it's for

- developers who want faster asset evaluation and integration
- publishers who want assets to be clearer, more reusable, and easier to adopt
- tool builders integrating marketplace discovery into IDEs, assistants, and pipelines
- AI-assisted workflows that need machine-readable asset metadata and implementation context

## Base path

All endpoints are served under:

`/api/v1`

## Current surface

The current API includes:

- agent search
- asset detail
- manifest, context, and integration endpoints
- capabilities taxonomy
- branch listing and version history
- entitlement checks
- publisher profiles

## How to navigate

Start with [Quickstart](/quickstart) if you want the shortest path from search to implementation. Use the generated Reference section when you need the full request and response surface by tag.

## Versioning

AssetArcade currently serves `/v1/`.

Breaking changes are introduced through a new major version rather than changing the live version in place. See the [Deprecation Policy](/deprecation-policy) for the compatibility rules.
