AssetArcade makes game assets easier to discover, evaluate, and integrate through structured metadata, compatibility information, version history, and implementation guidance. Instead of relying only on storefront copy, developers and tools can inspect the information they need to understand how an asset fits into a real project.

## Why AssetArcade exists

Modern game development increasingly happens inside tool-assisted workflows. AssetArcade exposes manifests, context, integration guidance, branches, versions, and publisher data in a machine-readable form so assets can move more cleanly from discovery to implementation. The goal is not to replace developers, but to make high-quality assets more usable inside the tools they already work with.

## Who it's for

- developers who want faster asset evaluation and integration
- publishers who want their assets to be more understandable and easier to adopt
- tool builders integrating marketplace discovery into IDEs, assistants, and pipelines
- developer-led, AI-assisted workflows that need structured asset metadata and implementation context

## Base path

All endpoints are served under:

`/api/v1`

## Current surface

The current API includes:

- agent search
- asset detail
- manifest, context, and integration endpoints
- capabilities taxonomy
- branch listing and version history
- entitlement checks
- publisher profiles

## How to navigate

Start with [Quickstart](/quickstart) if you want the shortest path from search to implementation. Use the generated Reference section when you need the full request and response surface by tag.

## Versioning

AssetArcade currently serves `/v1/`.

Breaking changes are introduced through a new major version rather than changing the live version in place. See the [Deprecation Policy](/deprecation-policy) for the compatibility rules.
