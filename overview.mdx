AssetArcade provides structured asset data for modern game development workflows. It helps developers, publishers, tools, and developer-led, AI-assisted workflows understand how an asset fits into a real project without relying only on storefront copy.

## What the API does

The API is built around a simple idea: assets should be easier to evaluate, easier to integrate, and easier to reason about in code.

With the current API, you can:

- search for assets by capability and basic constraints
- inspect full asset detail
- fetch focused metadata such as manifests, context, and integration guidance
- explore public branches and version history
- check authenticated user entitlement
- look up publisher profiles directly

## Why the API exists

Modern development workflows increasingly happen inside tools, assistants, and integration pipelines. To work well in those environments, assets need more than screenshots and marketing descriptions. They need structured metadata, compatibility details, implementation guidance, and version-aware context.

AssetArcade exposes that information in a structured, machine-readable format so high-quality assets can move more cleanly from discovery to implementation.

## How the API is organized

The current surface is grouped into a few focused areas:

- `Search` for capability-based asset discovery
- `Assets` for full detail and focused sub-resources
- `Capabilities` for taxonomy and filtering
- `Library` for entitlement checks
- `Publishers` for public publisher profiles

This keeps the API practical for both direct human use and tool-driven workflows.

## Base path and versioning

All current endpoints are served under:

`/api/v1`

The current API version is `v1`. Breaking changes are introduced through a new major version rather than changing the live version in place.

For the exact compatibility rules, see the [Deprecation Policy](/deprecation-policy).
